## Summary

This project examines how building energy performance in Chicago has changed since the introduction of the city’s Energy Rating system in 2019. The goal is to understand whether the public display of energy ratings is associated with improvements in energy efficiency and environmental outcomes over time. These ratings are intended to increase transparency and encourage building owners to reduce energy consumption, making it important to evaluate whether measurable changes have occurred since their implementation. The datasets begin in 2018 to demonstrate growth that is more likely caused by the implementation of the Energy Rating system.

We use multi-year Chicago Energy Benchmarking data from 2018–2019 through 2023–2024. These datasets are provided as separate annual files, each with slight differences in schema and formatting. We treat each year as a distinct data source and integrate them into a single unified panel dataset to enable longitudinal analysis. This approach allows us to track both overall trends and changes at the individual building level across multiple reporting periods.

The project follows a structured data workflow. First, we cleaned and standardized the raw datasets across years. This involved handling inconsistent column names, aligning schemas, and addressing missing values. We then merged all years into a unified dataset, adding a reporting_year field to support time-based analysis. A consistent building identifier was used to track buildings across years, allowing us to observe how specific properties change over time rather than only looking at aggregate statistics.

We performed both summary and longitudinal analysis. At the aggregate level, we computed yearly averages of energy use intensity (EUI), greenhouse gas emissions, and energy scores to observe a general, overall trends. At the building level, we calculated changes and percentage changes over time to understand how individual buildings have improved or declined. We also examined differences across property types to explore whether certain types of buildings show stronger improvements than others.

The results show a general decrease in both energy use and greenhouse gas emissions over time, particularly after 2020. This suggests that buildings may be becoming more efficient, although external factors such as changes in occupancy patterns or operational adjustments may also contribute to the decline seen throughout the datasets. We also observed variation across property types, indicating that building function plays a role in energy performance and how improvements are realized.

Overall, this project demonstrates how integrating multi-year benchmarking data can provide insight into trends in urban energy use and environmental impact. The workflow is fully reproducible, with scripts for data cleaning, analysis, and visualization, allowing others to replicate or extend the analysis. These results highlight the value of publicly available benchmarking data in understanding and monitoring changes in building performance over time.

## Data Profile

This project uses multi-year Chicago Energy Benchmarking data covering the reporting periods from 2018–2019 through 2023–2024. The data is publicly available from the City of Chicago data portal and includes building-level information on energy usage, emissions, and performance metrics. These datasets directly support the project’s goal of analyzing how energy efficiency and environmental outcomes have changed over time.

Although this project uses multiple years of the same dataset rather than separate external datasets, this approach was approved by the course instructor since each annual release functions as a distinct data source with differences in structure and content.

The raw datasets are stored in the repository under:
- `Data/Raw/2018-2019.csv`
- `Data/Raw/2019-2020.csv`
- `Data/Raw/2020-2021.csv`
- `Data/Raw/2021-2022.csv`
- `Data/Raw/2022-2023.csv`
- `Data/Raw/2023-2024.csv`

Each of these files represents a separate annual release of the benchmarking data. Although they contain similar types of information, the structure is not fully consistent across years. Column names vary, some fields are added or removed, and certain variables (such as ENERGY STAR scores) are not available in all years. Because of this, each file is treated as a distinct data source that requires cleaning and schema alignment before integration.

The datasets include key variables such as:
- Building identifiers
- Property name and address
- ZIP code and community area
- Primary property type
- Gross floor area
- Site Energy Use Intensity (EUI)
- Total greenhouse gas emissions
- Water usage
- ENERGY STAR score (when available)

After cleaning and integration, the processed dataset is saved as:
- `Data/Processed/energy_benchmarking_panel.csv`

In this final dataset, each row represents a building in a specific reporting year, allowing for longitudinal analysis across time. This structure enables a smoother, more comprehensible comparison both across years and across building types or locations, which is central to answering this project’s research questions.

There are minimal ethical or legal concerns associated with this dataset, as it is publicly released by the City of Chicago for transparency and research purposes. However, it is important to note that the data may contain missing or inconsistent values, and some buildings are exempt from reporting certain metrics, which can affect interpretation.

While additional contextual datasets (such as demographic or weather data) were considered during project planning, this project focuses on building a clean and reproducible integration of the multi-year benchmarking data. This ensures that the analysis remains consistent and fully reproducible within the scope of the available data.

NOTE: The Chicago Energy Benchmarking datasets used in this project are publicly available through the City of Chicago Data Portal (https://data.cityofchicago.org/). These datasets are subject to the City’s open data terms of use and are not owned by the authors of this project. The cleaned, integrated, and derived datasets produced in this project, along with all code (excluding the original raw data), are released under the Creative Commons Attribution 4.0 (CC BY 4.0) license.

## Data Quality

We assessed data quality using a Python profiling script (`Scripts/summarize_data.py`) that generates summary statistics, missing value counts, and dataset-level summaries for the integrated dataset. This script allows us to systematically identify potential issues such as missing data, inconsistent values, and uneven coverage across years.

One of the main issues in this dataset is missing data. Some variables, especially ENERGY STAR scores and water usage, are not available for all buildings or all years. In many cases, buildings are marked as exempt from reporting certain metrics, which results in structured missingness rather than random gaps. Because of this, missing values must be interpreted carefully, as they often reflect reporting rules rather than data errors. It is crucial to consider key factors that may affect not just the ENERGY STAR scores and water usage, but all variables, and make meaningful, statistic backed interpretations.

Another major challenge comes from inconsistencies across years. Since each annual dataset is released independently, the schema changes slightly over time. Column names are not always consistent, and some variables are introduced or removed in different years. For example, ENERGY STAR scores are only available in later datasets, and certain reporting fields appear or disappear depending on the reporting cycle. Without standardization, these differences would prevent meaningful comparisons across years.

We also observed inconsistencies in categorical variables. Fields such as primary property type and community area sometimes contain variations in naming or formatting (e.g., capitalization, spacing, or slight label differences). These inconsistencies can lead to duplicate categories or incorrect aggregations if not resolved during preprocessing.

In terms of dataset coverage, the number of reported buildings varies across years. This introduces an additional challenge when interpreting trends, since changes in averages may reflect differences in reporting participation rather than true changes in building performance. To account for this, we track the number of buildings per year and include this information in summary outputs to provide context for any observed trends.

Overall, the dataset is suitable for analysis but requires careful preprocessing. The primary data quality issues include structured missing values, schema inconsistencies across years, categorical inconsistencies, and variation in reporting coverage. These issues are addressed during the data cleaning stage through reproducible processing scripts, ensuring that the final dataset is consistent, interpretable, and suitable for longitudinal analysis.”

## Data Cleaning

Data cleaning was performed using a Python script (`Scripts/clean_and_merge.py`) to standardize and integrate the multi-year benchmarking datasets into a single consistent format. This step was necessary before any meaningful analysis could be done because each year’s dataset has slight differences in schema and formatting, which we have addressed in previous sections.

The first step in cleaning was standardizing column names across all datasets. Since different years use slightly different naming conventions, we mapped all columns to a consistent set of standardized names (e.g., aligning variations of energy use, emissions, and building identifiers). This directly addresses the schema inconsistencies identified in the data quality assessment and ensures that the same variables can be compared across years.

Next, we handled missing and inconsistent values. Certain fields, such as ENERGY STAR scores and water usage, are not available for all buildings or all years. Instead of removing these rows entirely, we preserved them and allowed missing values to remain as nulls. This decision reflects the structured nature of missingness in the dataset and avoids introducing bias by disproportionately removing buildings with incomplete reporting.

We also standardized data types across all datasets. Numeric fields such as energy use intensity (EUI) and greenhouse gas emissions were explicitly converted to numeric types, and invalid or non-numeric entries were handled during this process. This ensures consistency in downstream aggregation and prevents computation errors during analysis.

A key step in the process was combining all yearly datasets into a single integrated dataset. Each file was loaded, cleaned, and appended into a unified structure. A `reporting_year` column was added to preserve temporal context, allowing us to distinguish between observations from different reporting cycles and enabling longitudinal analysis.

To support tracking changes over time, the dataset was sorted by building identifier and reporting year. This structure allows for the computation of building-level changes and percentage differences across years, directly supporting the analytical goals of the project.

The final cleaned dataset was saved as:
- `Data/Processed/energy_benchmarking_panel.csv`

Additional derived datasets, such as building-level change metrics and aggregated summaries, were created in later stages of the workflow and stored in the `Data/Processed/` and `Results/Tables/` directories.

Overall, the cleaning process focuses on resolving schema inconsistencies, preserving structured missing data, standardizing variable formats, and enabling longitudinal tracking. These steps directly address the data quality issues identified earlier and ensure that the dataset is consistent, reproducible, and suitable for analysis.

## Findings

The analysis focuses on how building energy performance in Chicago has changed over time, particularly after the introduction of the Energy Rating system in 2019. Using the integrated multi-year dataset, we examined trends in energy use intensity (EUI), greenhouse gas emissions, and ENERGY STAR scores at both the aggregate and building level.

At the city-wide level, we observe a general downward trend in both average Site EUI and total greenhouse gas emissions over time. While there is some variation from year to year, both metrics show a noticeable decrease after 2020. This pattern is reflected in the yearly summary outputs generated in the analysis pipeline, where later reporting years consistently show lower average energy use compared to earlier ones. This suggests that buildings may be improving in terms of energy efficiency and environmental impact over the observed period.

However, these trends should be interpreted carefully. The decrease in energy use and emissions may not be solely attributable to the introduction of the Energy Rating system. External factors such as changes in occupancy, operational adjustments during COVID-19, or improvements in building technologies could also contribute to these patterns. Because of this, the observed trends represent associations rather than causal effects.

At the building level, the longitudinal analysis reveals that individual buildings experience varying levels of change over time. Using the derived dataset of building-level changes, we observe that some buildings show consistent decreases in EUI across multiple years, while others fluctuate or even increase. This variation highlights that improvements are not uniform and may depend on building-specific characteristics such as size, usage type, or management practices.

We also examined differences across property types. The results show that categories such as multifamily housing, offices, and schools make up a large portion of the dataset and heavily influence overall trends. When grouped by property type, there are clear differences in average energy use and emissions. For example, certain commercial or institutional buildings tend to have higher baseline energy usage compared to residential properties. This indicates that building function is an important factor when interpreting aggregate results.

ENERGY STAR scores provide an additional layer of insight into building performance. Buildings with higher scores generally correspond to lower energy use intensity, supporting the validity of this metric as an indicator of efficiency. However, missing values in earlier years limit the ability to fully track these scores over time, reducing their usefulness for long-term trend analysis.

Overall, the findings suggest that there has been a general improvement in energy performance across Chicago buildings over the observed period. At the same time, the results show that improvements are uneven across buildings and property types. While public reporting of energy metrics may be associated with positive trends, further analysis would be required to isolate the specific drivers of these changes.

## Future Work

There are several ways this project could be extended to provide deeper insight into building energy performance in Chicago. While the current work focuses on integrating and analyzing multi-year benchmarking data, additional data sources and analytical approaches could significantly expand the scope of the project.

One important extension would be the integration of external datasets. For example, demographic or socioeconomic data from sources such as the American Community Survey could be combined with the benchmarking data to explore equity-related questions. This could help answer whether certain neighborhoods experience different levels of energy efficiency or access to improvements. Similarly, incorporating weather data would allow for better control of external factors, since energy usage is often influenced by temperature and seasonal variation. These integrations would require careful alignment across geographic and temporal dimensions but would provide valuable additional context.

Another area for future work is the use of more advanced statistical or predictive modeling techniques. The current project focuses primarily on descriptive and longitudinal analysis. Future work could apply regression models or machine learning methods to better understand the factors driving changes in energy performance. For example, models could be used to predict energy use based on building characteristics or to estimate the impact of specific variables such as building size or property type.

Further improvements in data quality could also enhance the analysis. While this project addresses many issues related to schema inconsistencies and missing data, additional methods such as imputation or more advanced cleaning strategies could be applied. For example, missing ENERGY STAR scores or water usage values could potentially be estimated based on similar buildings or historical trends.

Another potential extension is more detailed spatial analysis. Although this project includes aggregation at the community area level, future work could explore geographic patterns more deeply using mapping or visualization tools. This could reveal spatial clusters of high or low energy performance and provide insight into how energy efficiency varies across different parts of the city.

Finally, future work could focus on evaluating policy impact more directly. While this project examines trends over time, it does not attempt to establish causality. More rigorous approaches, such as difference-in-differences analysis or controlled comparisons, could be used to assess whether the introduction of the Energy Rating system directly influenced building performance.

Overall, these extensions would build on the current reproducible pipeline while expanding both the depth and scope of the analysis.

## Challenges

One of the main challenges in this project was working with datasets that varied across years. Although the Chicago Energy Benchmarking data follows a similar structure annually, there are consistent differences in column names, available variables, and formatting. These inconsistencies required careful schema alignment before the datasets could be combined. Without standardization, it would not have been possible to perform reliable comparisons across years or construct a unified panel dataset.

Another major challenge was handling missing data. Several important variables are not available for all buildings or all years. In many cases, buildings are exempt from reporting certain metrics, resulting in structured missingness rather than random gaps. This made it necessary to design a cleaning approach that preserves as much usable data as possible while still maintaining analytical validity. Decisions about whether to keep or exclude missing values required careful consideration of how they would affect downstream analysis.

Tracking buildings across multiple years also presented difficulties. While the datasets include building identifiers, ensuring consistency across years required careful handling to avoid duplication or mismatches. Establishing a reliable identifier was critical for enabling longitudinal analysis, as even small inconsistencies could break the ability to track changes over time.

Another challenge involved balancing project scope with feasibility. The initial project plan included integrating additional external datasets, but this proved to be more complex than anticipated due to differences in format, coverage, and alignment. As a result, the project scope was adjusted to focus on building a strong, reproducible integration of the multi-year benchmarking data. While this reduced the breadth of the analysis, it allowed for a more robust and reliable pipeline.

Finally, interpreting the results presented its own challenges. Changes observed in energy use and emissions over time may be influenced by multiple factors, including reporting differences, external conditions, and changes in building operations. This makes it difficult to attribute trends to a single cause, such as the introduction of the Energy Rating system. As a result, care was taken to frame findings as associations rather than definitive conclusions.

Overall, these challenges highlight the complexity of working with real-world datasets and emphasize the importance of data cleaning, schema alignment, and thoughtful scoping in building a reproducible analysis pipeline.

### Reproducing

To reproduce this project:

1. Clone the repository and navigate into the project folder.

2. The raw datasets are already included in the repository under:
   `Data/Raw/`  
   (No additional data downloads are required.)

3. Install the required Python packages:

   ```bash
   pip install -r requirements.txt
   ```

4. Run the full pipeline:

   ```bash
   python Scripts/run_all.py
   ```

5. This script executes the complete workflow from raw data to final outputs (no manual steps required).
   - Clean and merge the raw datasets → `Data/Processed/energy_benchmarking_panel.csv`
   - Generate summary tables → `Results/Tables/`
   - Compute longitudinal changes → `Data/Processed/building_longitudinal_changes.csv`
   - Create visualizations → `Results/Figures/`

   <br>
6. (Optional) Verify data integrity:

   ```bash
   shasum -a 256 Data/Raw/*.csv
   ```

   Compare the output with the values stored in `checksums.txt` to confirm that the datasets match the originals.

7. Expected Outputs

   After running the pipeline, the following files will be generated:

   - Data/Processed/energy_benchmarking_panel.csv
   - Data/Processed/building_longitudinal_changes.csv
   - Results/Tables/
   - Results/Figures/
